# Запуск Nav2 в симуляции TurtleBot3

## Цель туториала
Запустить полный цикл автономной навигации в симуляции: загрузить карту, локализовать робота через AMCL и отправить его в целевую точку — сначала из RViz2, затем из терминала через действие `NavigateToPose`.

## Предварительные требования
* Установленная симуляция TurtleBot3 — [урок 10](<../lesson 10 (turtlebot3 sim)/README.md>).
* Установленный Nav2 — [Знакомство с Nav2](introducing_nav2.md).
* Сохранённая карта `~/map.yaml` — [Карта и локализация AMCL](nav2_localization.md).

---

## Шаг 1. Запуск мира Gazebo
В **первом терминале** запустите тот же мир, в котором строилась карта:

```bash
export TURTLEBOT3_MODEL=burger
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
```

## Шаг 2. Запуск Nav2
Во **втором терминале**:

```bash
export TURTLEBOT3_MODEL=burger
ros2 launch turtlebot3_navigation2 navigation2.launch.py use_sim_time:=True map:=$HOME/map.yaml
```

Этот launch-файл поднимает весь стек: `map_server` с вашей картой, `amcl`, `planner_server`, `controller_server`, `behavior_server`, `bt_navigator` и `lifecycle_manager`, а также открывает RViz2 с готовой конфигурацией.

Ожидаемый результат: в RViz2 отображается карта, но робот пока стоит «не на своём месте» или модель не отображается, а в терминале нет критических ошибок. Это нормально — AMCL ещё не знает, где робот.

## Шаг 3. Начальная оценка позы
1. Нажмите кнопку **2D Pose Estimate** в верхней панели RViz2.
2. Кликните на карте в точке, где робот стоит в Gazebo, и, не отпуская кнопку мыши, потяните стрелку в направлении «взгляда» робота.
3. Скан лидара (цветные точки) должен лечь на стены карты. Если не совпало — повторите.

Чтобы AMCL уточнила позу, можно немного подвигать робота телеуправлением в **третьем терминале**:

```bash
ros2 run turtlebot3_teleop teleop_keyboard
```

Облако зелёных стрелок (частицы AMCL) сожмётся вокруг робота. После этого **обязательно остановите teleop** (`Ctrl+C`): иначе он будет публиковать `cmd_vel` одновременно с контроллером Nav2 и робот будет дёргаться.

## Шаг 4. Отправка цели из RViz2
1. Нажмите кнопку **Nav2 Goal** в панели RViz2.
2. Кликните в свободную точку карты и потяните стрелку в направлении желаемой ориентации робота.

Робот немедленно поедет к цели: на карте появится глобальный маршрут, и контроллер поведёт робота вдоль него, объезжая колонны. Если на пути поставить препятствие, локальная costmap его увидит, и маршрут перестроится.

## Шаг 5. Отправка цели из терминала
Навигация — это действие `NavigateToPose` (см. [урок 3](<../lesson 3 (actions and services)/understanding_actions.md>)). Убедимся, что action-сервер запущен:

```bash
ros2 action list | grep navigate_to_pose
```

Отправим цель из терминала с обратной связью:

```bash
ros2 action send_goal /navigate_to_pose nav2_msgs/action/NavigateToPose \
  "{pose: {header: {frame_id: map}, pose: {position: {x: 0.5, y: 0.5}, orientation: {w: 1.0}}}}" \
  --feedback
```

Ожидаемый результат: терминал печатает feedback (оставшееся расстояние, прошедшее время), робот едет в точку `(0.5, 0.5)` фрейма `map`, по завершении приходит статус `SUCCEEDED`.

Так навигацию можно вызывать из собственных нод — это обычный action-клиент, как в уроке 3.

## Контрольные проверки
Все ноды стека запущены:

```bash
ros2 node list
```

В списке присутствуют `/amcl`, `/bt_navigator`, `/controller_server`, `/planner_server`, `/behavior_server`, `/map_server`, `/lifecycle_manager_navigation`.

Цепочка tf-фреймов собрана:

```bash
ros2 run tf2_ros tf2_echo map base_link
```

Команда печатает текущую позу робота на карте (Translation / Rotation) — значит, цепочка `map -> odom -> base_link` опубликована полностью.

Lifecycle-ноды активны:

```bash
ros2 lifecycle get /amcl
```

```
active [3]
```

## Диагностика

| Симптом | Причина и решение |
|---|---|
| Робот не реагирует на Nav2 Goal | Не задана начальная поза — выполните 2D Pose Estimate (шаг 3) |
| Скан лидара не совпадает с картой | Карта строилась в другом мире, либо поза задана неточно — повторите оценку позы |
| В RViz2 ошибки tf «message filter dropping» | Забыт `use_sim_time:=True` — перезапустите Nav2 с этим параметром |
| Робот дёргается или едет рывками | Параллельно запущен teleop — остановите его |
| `map_server` падает при старте | Неверный путь в `map:=` или в `map.yaml` поле `image` указывает не на тот файл |

## Итог
* Полный цикл навигации: мир Gazebo → `navigation2.launch.py` с картой → начальная поза → цель.
* Цель можно задавать из RViz2 (Nav2 Goal) или программно через действие `NavigateToPose`.
* Ключевые проверки: список нод, `tf2_echo map base_link`, состояние lifecycle-нод.

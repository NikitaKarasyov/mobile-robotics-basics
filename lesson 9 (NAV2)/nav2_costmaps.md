# Costmap: как Nav2 видит препятствия

## Цель туториала
Понять, что такое **costmap**, чем глобальная costmap отличается от локальной, из каких слоёв они собираются и как проверить их работу через топики. Эти знания понадобятся при запуске навигации в симуляции и при настройке параметров под своего робота.

---

## Что такое costmap?
**Costmap (карта стоимостей)** — это сетка из ячеек, где каждой ячейке присвоена «стоимость» проезда: от 0 (свободно) до 254 (гарантированное столкновение). Значение 255 означает «нет данных».

Планировщик и контроллер не работают с «сырой» картой и сырыми данными лидара напрямую — они работают с costmap. Маршрут строится так, чтобы суммарная стоимость пути была минимальной, а ячейки со стоимостью столкновения объезжались.

Ключевые значения стоимости:

| Стоимость | Смысл |
|---:|---|
| 0 | Свободное пространство |
| 1–252 | Чем выше, тем ближе к препятствию |
| 253 | Возможное столкновение (зона рядом с препятствием) |
| 254 | Гарантированное столкновение (само препятствие) |
| 255 | Неизвестная область |

## Глобальная и локальная costmap
В Nav2 одновременно работают две costmap:

| | Глобальная costmap | Локальная costmap |
|---|---|---|
| Кем используется | `planner_server` (маршрут до цели) | `controller_server` (объезд препятствий рядом) |
| Покрытие | Вся карта | Небольшое окно вокруг робота (например, 3x3 м), движется вместе с ним |
| Основа | Статическая карта + сенсоры | Только сенсоры |
| Фрейм | `map` | `odom` |

Логика разделения: глобальный маршрут строится по всей известной карте, а реагировать на внезапные препятствия (человек прошёл перед роботом) нужно быстро и только в ближней зоне — для этого достаточно маленькой локальной costmap, которая обновляется с высокой частотой.

---

## Слои costmap
Costmap собирается из **слоёв (layers)**, каждый слой добавляет свою информацию:

* **Static Layer** — берёт ячейки из статической карты, опубликованной `map_server` (стены, мебель — всё, что было на карте при её построении). Используется в глобальной costmap.
* **Obstacle Layer** — отмечает препятствия по данным сенсоров (топик `scan`) и очищает ячейки, через которые луч лидара прошёл свободно.
* **Inflation Layer** — «раздувает» препятствия: добавляет вокруг каждого препятствия зону убывающей стоимости. Благодаря этому маршрут прокладывается с запасом от стен, а не впритирку.

Слой инфляции — главный инструмент настройки «осторожности» робота: чем больше радиус инфляции, тем дальше робот держится от препятствий.

---

## Costmap и lifecycle
Обе costmap живут внутри lifecycle-нод `planner_server` и `controller_server` и проходят те же состояния (`unconfigured -> inactive -> active`), что и остальные серверы Nav2. Пока `lifecycle_manager` не активировал стек, costmap не публикуются.

## Контрольные проверки
Проверки выполняются при запущенном стеке Nav2 (запуск в симуляции разобран в статье [Запуск Nav2 в симуляции TurtleBot3](nav2_turtlebot3_sim.md); вернитесь к этим командам оттуда).

Список топиков costmap:

```bash
ros2 topic list | grep costmap
```

Ожидаемый результат — топики обеих costmap:

```
/global_costmap/costmap
/global_costmap/costmap_updates
/local_costmap/costmap
/local_costmap/costmap_updates
```

Тип сообщения — та же занятостная сетка, что и у обычной карты:

```bash
ros2 topic info /global_costmap/costmap
```

```
Type: nav_msgs/msg/OccupancyGrid
```

Метаданные глобальной costmap (размер, разрешение, фрейм):

```bash
ros2 topic echo /global_costmap/costmap --once --field info
```

В RViz2 costmap добавляется как display типа **Map** с топиком `/global_costmap/costmap` (цветовая схема costmap). Вы увидите карту, где вокруг стен есть цветная зона инфляции.

---

## Итог
* Costmap — сетка стоимостей, по которой Nav2 планирует и контролирует движение.
* Глобальная costmap покрывает всю карту и служит планировщику; локальная — окно вокруг робота для контроллера.
* Costmap собирается из слоёв: статическая карта, препятствия от сенсоров, инфляция.

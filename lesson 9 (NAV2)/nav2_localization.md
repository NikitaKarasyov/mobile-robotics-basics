# Карта и локализация AMCL

## Цель туториала
Построить карту мира TurtleBot3 в симуляции с помощью SLAM, сохранить её в файлы `map.pgm` + `map.yaml` и понять, как **AMCL** локализует робота на готовой карте. Полученная карта понадобится в следующей статье для запуска Nav2.

Перед началом должна быть установлена симуляция TurtleBot3 — см. [урок 10](<../lesson 10 (turtlebot3 sim)/README.md>).

---

## SLAM и локализация: в чём разница?
Навигация по карте состоит из двух отдельных задач:

* **SLAM** (Simultaneous Localization and Mapping) — робот **строит карту** неизвестного окружения и одновременно отслеживает своё положение в ней. Выполняется один раз, «экскурсией» по помещению.
* **Локализация** — карта уже есть, и робот определяет, **где он на ней находится**, сравнивая данные лидара с картой. Выполняется постоянно во время навигации.

В этом туториале мы сначала построим карту (SLAM, пакет `turtlebot3_cartographer`), а затем разберём, как работает локализация (AMCL).

## Шаг 1. Построение карты в симуляции

### Запуск мира Gazebo
В **первом терминале**:

```bash
export TURTLEBOT3_MODEL=burger
ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py
```

Откроется Gazebo с роботом внутри шестиугольной арены с колоннами.

### Запуск SLAM-ноды
Во **втором терминале** запустите Cartographer — SLAM-метод TurtleBot3 по умолчанию:

```bash
export TURTLEBOT3_MODEL=burger
ros2 launch turtlebot3_cartographer cartographer.launch.py use_sim_time:=True
```

Параметр `use_sim_time:=True` обязателен: в симуляции все ноды должны использовать симуляционное время из топика `/clock`, а не системные часы, иначе трансформации tf «разъедутся» по времени.

Откроется RViz2, в котором карта начнёт строиться вокруг робота: серое — свободное пространство, чёрное — препятствия, тёмное поле вокруг — ещё не исследованная область.

### Телеуправление
В **третьем терминале** запустите телеуправление и объедьте арену так, чтобы лидар «увидел» все стены и колонны:

```bash
export TURTLEBOT3_MODEL=burger
ros2 run turtlebot3_teleop teleop_keyboard
```

Двигайтесь медленно и плавно: резкие повороты ухудшают качество карты.

### Сохранение карты
Когда карта в RViz2 выглядит полной (все стены замкнуты, «дыр» нет), сохраните её в **четвёртом терминале**:

```bash
ros2 run nav2_map_server map_saver_cli -f ~/map
```

Ожидаемый результат — два файла в домашнем каталоге:

```
map.pgm    # изображение карты (занятостная сетка)
map.yaml   # метаданные карты
```

После сохранения SLAM-ноду и teleop можно остановить (`Ctrl+C`).

## Шаг 2. Что внутри файлов карты?
`map.pgm` — обычное изображение в градациях серого, его можно открыть любым просмотрщиком. `map.yaml` описывает, как это изображение привязано к миру:

```yaml
image: map.pgm
mode: trinary
resolution: 0.05
origin: [-2.34, -1.85, 0]
negate: 0
occupied_thresh: 0.65
free_thresh: 0.25
```

* `resolution` — размер одной ячейки в метрах (здесь 5 см);
* `origin` — координаты левого нижнего угла изображения во фрейме `map`;
* `occupied_thresh` / `free_thresh` — пороги, по которым пиксель считается занятым или свободным.

Именно эту пару файлов загружает `map_server` при старте Nav2.

## Шаг 3. Как работает AMCL?
**AMCL** (Adaptive Monte Carlo Localization) — нода локализации Nav2. Она использует **фильтр частиц**:

1. По карте рассеивается множество **частиц** — гипотез «робот может быть здесь, с такой ориентацией».
2. При движении робота все частицы смещаются по данным одометрии.
3. Каждое измерение лидара сравнивается с картой для каждой частицы: гипотезы, в которых скан «совпал» со стенами, получают больший вес.
4. Частицы с малым весом отбрасываются, с большим — размножаются. Облако частиц постепенно сжимается вокруг истинного положения робота.

Результат работы AMCL — трансформация `map -> odom` (см. [Знакомство с Nav2](introducing_nav2.md)): она исправляет накопившийся дрейф одометрии, чтобы положение робота на карте оставалось верным.

Практическое следствие: AMCL нужна **начальная оценка позы** (initial pose). Пока вы не указали, где примерно стоит робот, частицы рассеяны по всей карте и локализация не работает. В RViz2 для этого служит кнопка **2D Pose Estimate** — мы воспользуемся ею в следующей статье.

## Контрольные проверки
Файлы карты на месте и корректны:

```bash
ls -l ~/map.pgm ~/map.yaml
cat ~/map.yaml
```

В `map.yaml` поле `image` указывает на существующий `map.pgm`, `resolution` равно `0.05`.

Пакет AMCL установлен:

```bash
ros2 pkg list | grep nav2_amcl
```

## Итог
* Карта строится один раз через SLAM (Cartographer) и сохраняется парой файлов `map.pgm` + `map.yaml`.
* AMCL локализует робота на готовой карте фильтром частиц и публикует трансформацию `map -> odom`.
* Для старта AMCL требуется начальная оценка позы.
